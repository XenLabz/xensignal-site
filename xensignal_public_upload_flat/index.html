What XenSignal does

XenSignal evaluates candidate entries using model fair value, market pricing, spread, timing, position state, and optimizer guidance. The current public-facing build is designed to present evidence, not execute trades.

Why shadow mode matters

Shadow mode lets the system observe, score, and log candidate trades without blocking or placing real orders. In other words, the system gets to prove itself before touching money like a civilized creature.

Current operating posture

Paper-watch only. Engine consumption disabled.

The optimizer can recommend profiles and filters, but live execution blocking is intentionally not enabled. Candidate decisions can be evaluated and logged while keeping the deployment safely observational.

1. Local engine

XenSignal runs locally or in a controlled compute environment where the model, logs, and market data pipeline live.

2. Static exports

The engine exports JSON summaries and report files. cPanel hosts those files like a useful filing cabinet, not a pretend GPU cluster.

3. Hosted viewer

The XenLabz site displays dashboards, status, and reports from static files using HTML, CSS, and JavaScript.
